Porquê usamos Nginx e Ansible

Foi escolhido o utilizar Nginx, por duas razões: Facilidade de configuração e para o aprendizado de novas tecnologias além de possuir muitas vantagens como, Facilidade e rapidez para armazenar arquivos estáticos, que são arquivos tipo zip, xml, mp4 podendo ser trabalhado com maior performance no Nginx, pelo motivo no qual ele suporta o armazenamento em cache, fazendo com que o processo seja mais ágil.
O Ansible tem uma curva de aprendizado muito rápida, sendo fácil a instalação e configuração inicial. A sintaxe e o fluxo de trabalho são de simples de compreensão, facilitando o aprendizado de novos usuários. Os arquivos utilizam um padrão YAML, um padrão de linguagem declarativa de dados amigável, que é amplamente utilizado por outras ferramentas e de fácil entendimento.

Pre-requisitos para provisionamento com vagrant + ansible

  1. Insale o vagrant usando o comando "apt install vagrant".
  2. Instale o virtualbox usando o comando "apt install virtualbox".
  3. Instale o ansible usando o comando "apt install ansible".

Passos para provisionar uma box usando vagrant com ansible

  1. Crie um vagrant file com as configurações iniciais da box desejada e defina que o provisionamento será feito com o ansible e defina o playbook utilizando o seguinte comando: (box).vm.provision "ansible", playbook: "provisioning/nginx.yml"
  2. Crie o playbook para ser utilizado na box utilizando a extensão .yml como no exemplo mostrado nas configurações abaixo.
  3. Levante a box utilizando o comando vagrant up.
